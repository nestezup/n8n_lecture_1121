## 자동화 = 규칙이 있는 반복 작업을 컴퓨터에게 맡기는 것

**예시: 이메일 → 슬랙 알림**

수동으로 할 때:
1. 이메일 열기
2. 제목에 "긴급" 있는지 확인
3. 슬랙 열기
4. 메시지 작성해서 전송

자동화 후:
→ 컴퓨터가 1초 만에 처리

## n8n이란?

**블록 조립하듯 자동화 만드는 도구**

- 6814개 이상의 앱/서비스 연동
- 코딩 불필요
- 무료 오픈소스
- 클라우드/셀프호스팅 모두 가능

**오늘은 클라우드 서버로 진행합니다**

---

## n8n의 3가지 핵심 개념

**1. 워크플로우 (Workflow)**
- 자동화의 전체 시나리오
- 하나의 완성된 자동화 프로세스

**2. 노드 (Node)**
- 각각의 작업 단위 (레고 블록처럼 조립)
- 데이터 입력, 변환, 출력 등 개별 기능 수행

**3. 트리거 (Trigger)**
- 자동화를 시작시키는 신호
- 시간, 이벤트, 웹훅 등 다양한 방식

---

## 모든 워크플로우의 기본 구조

```
[트리거] → [작업] → [작업] → [결과]
```

**구체적 예시:**
```
[매일 9시] → [날씨 확인] → [메시지 작성] → [이메일 전송]
```
```
[폼 제출] → [데이터 정리] → [저장] → [알림]
```

이 구조는 모든 자동화의 기본이 됩니다.

## n8n 화면 구성

**왼쪽 패널**: 노드 라이브러리
- 사용 가능한 모든 노드 목록
- 검색으로 필요한 노드를 빠르게 찾을 수 있음

**중앙 캔버스**: 워크플로우 작업 공간
- 노드를 배치하고 연결하는 메인 작업 영역
- 드래그 앤 드롭으로 노드 배치

**오른쪽 패널**: 노드 설정
- 선택한 노드의 상세 설정 입력
- 파라미터, 인증 정보 등 구성

**상단 바**: 실행/저장 버튼
- Execute Workflow: 수동 실행
- Save: 워크플로우 저장
- Active: 자동 실행 활성화

**각자 2분간 화면을 둘러보고 메뉴 위치를 확인하세요**

## Code 노드 이해하기

**언제 사용하나요?**
- 복잡한 계산이 필요할 때
- 데이터 형식을 변환해야 할 때
- 조건부 로직을 구현할 때
- 기존 노드로 구현하기 어려운 작업

**JavaScript를 모른다면?**
- ChatGPT나 Claude에게 요청하면 코드를 작성해줍니다
- 대부분의 일반적인 로직은 기존 노드만으로도 충분합니다
- 간단한 수정은 예제를 참고하여 가능합니다

**주요 문법:**
- `$input.item.json.필드명`: 이전 노드의 데이터 접근
- `return { json: {...} }`: 다음 노드로 데이터 전달

## IF 노드의 활용

```
[데이터] → [조건 확인]
              ├─ 조건 충족 → [액션 A]
              └─ 조건 불충족 → [액션 B]
```

**실전 활용 사례:**
- 거래 금액이 일정 금액 이상이면 관리자에게 알림
- 업무 시간인지 확인하여 다르게 처리
- 특정 키워드가 포함되어 있으면 분류
- 재고가 부족하면 발주 프로세스 시작

## 실습 5 - Step 3: 조건 설정

**Mode**: `Rules`

**Rule 1** (A등급):
- Value 1: `{{ $json.score }}`
- Operation: `Larger Than or Equal`
- Value 2: `90`
- Output: `0`

**Rule 2** (B등급):
- Value 1: `{{ $json.score }}`
- Operation: `Larger Than or Equal`
- Value 2: `80`
- Output: `1`

**Rule 3** (C등급):
- Value 1: `{{ $json.score }}`
- Operation: `Larger Than or Equal`
- Value 2: `70`
- Output: `2`

**Rule 4** (D등급):
- Fallback: 위 조건에 해당하지 않는 모든 경우
- Output: `3`

Switch는 위에서 아래로 조건을 검사하며, 처음 만족하는 조건의 출력으로 데이터를 보냅니다.

## Switch vs IF 비교

**IF 노드**
- 용도: 2개 경로 분기 (예/아니오)
- 조건: 단일 조건 평가
- 사용 예시: 승인/거부, 정상/오류, 주중/주말

**Switch 노드**
- 용도: 3개 이상 경로 분기
- 조건: 여러 조건을 순차적으로 평가
- 사용 예시: 등급 분류, 우선순위 분류, 상태별 처리

**실전 활용:**
- 고객 유형별로 다른 메시지 전송
- 주문 금액에 따른 배송비 계산
- 업무 중요도에 따른 담당자 배정

## API 이해하기

**API (Application Programming Interface)**

## 표현식 {{ }} 이해하기

**표현식의 구조**
- `{{ $json.필드명 }}`: 이전 노드의 데이터 참조
- `{{ $json.main.temp }}`: 계층 구조 접근 (점 표기법)
- `{{ $json.weather[0].description }}`: 배열 접근 (인덱스 사용)

**주요 변수:**
- `$json`: 이전 노드의 JSON 출력 데이터
- `$input`: 전체 입력 데이터 (여러 항목 포함 가능)
- `$node`: 특정 노드의 데이터 참조

**입력 방법:**
- 직접 입력: 정확한 경로를 알고 있을 때
- 드래그 앤 드롭: 실수를 방지하고 정확한 경로 자동 생성

## 실습 8: 웹훅으로 외부 연동

## 웹훅(Webhook) 이해하기

**일반 트리거 (Polling)**
- n8n이 주기적으로 외부 서비스 확인
- 예: 1분마다 구글 시트 확인

**웹훅 (Webhook)**
- 외부 서비스가 n8n에게 직접 알림
- 이벤트 발생 즉시 처리 가능

**웹훅 사용 사례:**
- 구글 폼 제출 → n8n 즉시 처리
- 결제 완료 → 자동 처리 시작
- 외부 시스템 이벤트 → n8n 연동
- 다른 자동화 도구 → n8n 연결

**장점:** 실시간 처리, 불필요한 반복 확인 없음

## 구글 시트 활용 방안

**간단한 데이터베이스 역할**
- 소규모 데이터 저장 및 관리
- 팀원들과 쉽게 공유 가능
- 별도 데이터베이스 서버 불필요

**실전 조합 패턴:**
- 웹훅으로 데이터 수신 → 시트에 자동 저장
- 폼 제출 내용 → 시트에 기록
- 정기 리포트 생성 → 시트에 작성

**주의사항:**
- 대용량 데이터(수만 행 이상)는 실제 데이터베이스 권장
- 동시 쓰기 작업이 많으면 충돌 가능

## 흔한 에러와 해결법

**"Credentials invalid"**
- 원인: 인증 정보 만료 또는 권한 부족
- 해결: Credential 재인증

**"Node not found"**
- 원인: 노드 이름 오타 또는 삭제된 노드 참조
- 해결: 노드 이름 확인, 연결 재설정

**"Expression error"**
- 원인: {{ }} 표현식 문법 오류 또는 존재하지 않는 필드 참조
- 해결: 표현식 문법 확인, 이전 노드 데이터 구조 점검

**"Rate limit exceeded"**
- 원인: API 호출 횟수 제한 초과
- 해결: 호출 간격 늘리기, API 플랜 확인

**"Timeout"**
- 원인: 응답 시간 초과 (느린 API, 네트워크 문제)
- 해결: 타임아웃 설정 증가, API 상태 확인

## 과제 (선택사항)

**다음 수업까지:**

1. 오늘 만든 워크플로우 중 1개를 실제 업무에 적용해보기
2. 템플릿 라이브러리에서 2개 이상 가져와서 분석
3. 자신의 업무 자동화 아이디어 3개 구체화 (어떤 노드를 사용할지 계획)
4. 가능하면 간단한 자동화 1개 완성

다음 수업에서 경험을 공유하는 시간을 가지겠습니다.
